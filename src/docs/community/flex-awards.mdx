## How It Works

1. **Kickoff Meeting** — A meeting is held to introduce the FLEX Awards to all teams, explain the categories, evaluation criteria, and nomination process.
2. **Nomination Window** — A form is circulated via email for team leads and peers to nominate candidates or self-nominate. Nominees provide supporting evidence such as slides, videos, and written descriptions.
3. **Evaluation** — The Brightlayer UI team evaluates all submissions against the defined criteria for each category. Winners are notified prior to the public announcement.
4. **Award Ceremony** — A dedicated meeting is held at the Global Experience Design Summit to announce and celebrate the winners.

## Award Categories

### 1. Contributor Award

Recognizes individuals who have actively contributed to evolving the Brightlayer UI design system.

**Evaluation Criteria:**

| Criterion | Weight | Description |
| --- | --- | --- |
| Participation | 40% | How frequent was the contribution, and how much effort was spent. |
| Impact | 40% | How much business and community influence the contribution has made or will make. |
| Platforming & Scale | 20% | Whether this is a one-off contribution or a pattern that can be scaled and extended to fit other projects' needs. |

<Divider />

### 2. Community Award

Recognizes individuals who have gone above and beyond to help other teams adopt and succeed with Brightlayer UI.

**Evaluation Criteria:**

| Criterion | Weight | Description |
| --- | --- | --- |
| Participation | 40% | How frequent was the contribution, and how much effort was spent. |
| Community Impact | 60% | How much community influence the contribution has made or will make. |

<Divider />

### 3. Learner Award

Recognizes individuals who have acquired new UI skills (design or development) and applied them effectively in their work.

**Evaluation Criteria:**

| Criterion | Weight | Description |
| --- | --- | --- |
| Application | 60% | How has the learning been used to improve the nominee's work? |
| Sharing | 25% | How has the nominee inspired others to learn? How did they share their new knowledge or skill with others? |
| Outside the Comfort Zone | 15% | How different is this learning topic from the nominee's everyday work? |

<Divider />

### 4. Execution Award

Recognizes products and teams that have best followed the Brightlayer UI design and development standards in their applications.

The Execution Award is scored across two major dimensions:

#### Design Evaluation (40%)

Assesses visual adherence to Brightlayer UI standards (UX is not evaluated, only UI):

| Criterion | Weight | Description |
| --- | --- | --- |
| Components (Choice & Alignment) | 30% | Proper selection and alignment of Brightlayer UI components. |
| Animations | 5% | Quality and appropriateness of UI animations. |
| Colors / Accessibility | 20% | Correct use of the Brightlayer UI color palette and accessibility compliance. |
| Icons | 20% | Proper use of Brightlayer UI iconography. |
| Information Density | 10% | Effective use of space and information layout. |
| New Inventions / Breaking Rules | 15% | Creative deviations from standard patterns where justified. |

#### Development Evaluation (60%)

Checks adherence to Brightlayer UI coding best practices:

| Criterion | Description |
| --- | --- |
| Unnecessary Override | Whether the product avoids unnecessary overrides of Brightlayer UI defaults. |
| Encapsulation | Proper encapsulation and code organization. |
| Documentation | Quality of project documentation (README, comments, setup instructions). |
| BLUI Packages | Use of official Brightlayer UI packages (themes, colors, components). |
| Up-to-date | Whether dependencies and Brightlayer UI packages are on current versions. |

## How to Participate

- **Nominate yourself or a peer** when the nomination window opens.
- **Provide supporting materials** — Include slides, videos, or code links to strengthen your nomination.
- **Attend the kickoff meeting** — Learn about the current year's categories and expectations.

For questions about the FLEX Awards, please [contact the Brightlayer UI team](/community/contactus).
